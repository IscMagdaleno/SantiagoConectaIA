Para futuras referencias y tareas de mantenimiento, aquí tienes las URLs críticas que utilizamos para gestionar la configuración de tu proyecto **Agente Santaigo**:

* **Panel de control de la aplicación (Principal):**
[https://developers.facebook.com/](https://developers.facebook.com/)
Aquí es donde seleccionas tu app "Agente Santaigo" para acceder a la configuración de productos, webhooks y registros.
* **Configuración de la API de WhatsApp (Webhooks y Registro de números):**
[https://developers.facebook.com/apps/](https://developers.facebook.com/apps/)
Dentro de tu app, navega a **WhatsApp > Configuración de la API**. Esta es la URL donde obtienes el *Phone Number ID*, el *WABA ID* y donde configuras la URL de devolución de llamada para tu API en Azure.
* **Configuración básica de la App (App Secret):**
[https://developers.facebook.com/apps/](https://developers.facebook.com/apps/)
Dentro de tu app, navega a **Configuración > Básica**. Aquí es donde obtienes el *App Secret* necesario para la autenticación de tus peticiones.
* **Gestión del Portafolio Comercial ("Santiago Agente"):**
[https://business.facebook.com/settings/info](https://business.facebook.com/settings/info)
Aquí puedes administrar los detalles legales, dirección y el estado de verificación de tu portafolio comercial vinculado.
* **Documentación oficial de la API de WhatsApp (para referencia de errores):**
[https://developers.facebook.com/docs/whatsapp/cloud-api/support/error-codes/](https://developers.facebook.com/docs/whatsapp/cloud-api/support/error-codes/)
Este enlace fue fundamental para diagnosticar los códigos de error como el `131047` (Re-engagement) y el `100` (GraphMethodException) durante las pruebas.
